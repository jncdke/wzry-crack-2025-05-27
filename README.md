# 安卓内核模块集合 (Android Kernel Module Collection)

## 项目简介 (Project Introduction)

本项目包含一系列为安卓设备定制的内核模块 (.ko 文件) 及相关的加载脚本 (.sh 文件)。这些模块的主要目的是对系统进行修改，可能用于性能调优（例如游戏优化，基于 "GT" 目录推测）、进程隐藏（基于 "新方案隐藏进程" 描述推测）或其他底层系统调整。

**请注意：**
*   刷入内核模块属于高级操作，需要用户对安卓系统和 Linux 内核有一定了解。
*   **必须拥有 ROOT 权限才能加载这些内核模块。**
*   错误地使用或刷入不兼容的模块可能会导致系统不稳定、功能异常甚至无法启动。请务必谨慎操作，并自行承担风险。

本 README旨在提供一个中文的详细说明，帮助用户了解项目内容和基本使用方法。

## 目录结构说明 (Directory Structure Explanation)

本项目的目录结构主要根据模块类型、内核版本和特定设备进行组织：

*   **顶层目录 (Top-Level Directories):**
    *   `GT/`, `GT V1.6-alpha/`, `GT V2.0/`: 这些目录下的模块可能与游戏性能优化有关 (推测 "GT" 为 "Game Turbo" 或类似概念)。不同版本可能代表不同时期的优化方案或功能集。
    *   `RT/`, `RT驱动/`: 这些目录下的模块可能与实时性能或特定驱动程序相关 (推测 "RT" 为 "Real-Time")。`RT驱动` 可能是 `RT` 模块的另一种组织方式或更新。
    *   `rt proc/`: 此目录下的模块可能专注于进程相关的实时调整或管理。

*   **内核版本子目录 (Kernel Version Subdirectories):**
    *   在上述各类型目录下，通常会包含以内核版本号命名的子目录，例如 `4.14.117/`, `4.19.157/`, `5.10/`, `6.1/` 等。用户需要根据自己设备的内核版本选择对应的模块。
    *   文件名中带有 `(新方案隐藏进程)` 的目录，例如 `GT/4.14.117(新方案隐藏进程)/`，表明其中的模块包含了隐藏进程的功能。

*   **特定设备/品牌标识 (Device/Brand Specific Identifiers):**
    *   部分目录或文件名可能包含特定设备或品牌信息，例如 `oneplus` (一加), `xiaomi` (小米), `oppo` (OPPO), `ColorOS`, 或特定型号如 `note12pro`。这表示该模块是为这些特定设备或系统版本定制或测试过的。

*   **核心文件 (Core Files):**
    *   `.ko` 文件: 这是预编译的 Linux 内核模块文件。
    *   `load.sh`: 这是用于加载对应 `.ko` 模块的shell脚本。通常需要 ROOT 权限执行。
    *   `说明(第一次刷驱动必看...).txt`: 提供基本刷入说明的文本文件。
    *   `检查驱动报错.sh`: 用于检查内核日志中与模块相关的错误信息 (通常是 `dmesg | grep hack`)。
    *   `王者荣耀.sh`: 针对特定游戏《王者荣耀》的脚本，具体功能需查看其内容。

用户在选择模块时，应综合考虑模块类型 (GT, RT等)、自己设备的安卓内核版本以及是否有针对自己设备的特别说明。

## 使用方法 (Usage Guide)

请严格按照以下步骤操作，并确保您已了解相关风险。

**步骤 1: 确认设备内核版本并选择模块**

1.  **查看内核版本:**
    *   通常可以在设备的 "设置" (Settings) > "关于手机" (About phone) > "软件信息" (Software information) 或类似路径下找到 "内核版本" (Kernel version)。例如 `4.14.117-perf-g7a1abd6`，其中 `4.14.117` 是主要版本号。
2.  **选择模块:**
    *   根据您的内核版本，在本项目中找到对应的目录 (例如 `GT/4.14.117(新方案隐藏进程)/`)。
    *   如果您使用的是特定品牌手机 (如 OPPO, Realme, OnePlus, Xiaomi)，请优先选择有对应品牌标识的目录或模块。
    *   注意模块的功能说明，例如是否包含 `(新方案隐藏进程)`。

**步骤 2: 传输文件到设备**

1.  将选定的 `.ko` 内核模块文件和对应的 `load.sh` 脚本下载到您的电脑。
2.  通过 USB 数据线或其他方式，将这两个文件传输到您安卓设备的内部存储中，建议放在一个容易查找的目录下 (例如，创建一个新目录 `MyKernelModules`)。

**步骤 3: 执行加载脚本 (需要 ROOT 权限)**

1.  在您的安卓设备上打开一个终端模拟器应用 (例如 Termux, MT管理器等)。
2.  **获取 ROOT 权限:** 在终端中输入 `su` 命令并授予权限。
3.  **导航到文件目录:** 使用 `cd` 命令切换到您存放 `.ko` 和 `load.sh` 文件的目录。例如: `cd /sdcard/MyKernelModules/`
4.  **授予执行权限 (如果需要):** `chmod +x load.sh`
5.  **执行加载脚本:** 输入 `sh load.sh` 或 `./load.sh` 并回车。

**步骤 4: 验证与故障排除**

1.  **查看脚本输出:**
    *   **成功标志:**
        *   根据 `GT/说明(第一次刷驱动必看...).txt` 文件，如果脚本执行后直接结束进程且没有任何提示，则表示刷入成功。
        *   部分 `load.sh` 脚本在成功时会输出绿色提示信息，如 "操作成功: 内核模块已经成功加载。"
    *   **失败标志:**
        *   脚本输出红色错误信息，例如 "刷入失败，请重启手机后再试一次..."。
        *   脚本提示 "Invalid argument" (无效参数)。
        *   部分 `load.sh` 脚本在失败后可能会尝试自动重启设备。
2.  **检查内核日志 (推荐):**
    *   如果怀疑模块加载失败或工作不正常，可以使用项目中的 `检查驱动报错.sh` 脚本 (内容通常是 `dmesg | grep hack`)，或者直接在已获取 ROOT 权限的终端中执行 `dmesg | grep hack` 或 `dmesg | tail -n 50` (查看最后50行日志)。
    *   留意任何与您尝试加载的模块相关的错误信息。
3.  **故障排除提示 (来自 `load.sh` 和经验):**
    *   **"Invalid argument":** 尝试再次执行 `load.sh`。
    *   **OPPO, Realme, OnePlus 设备:** 可能需要通过签名验证，并且建议升级到安卓13或更高版本。具体操作可能需要查阅相关社区。
    *   **长时间未重启设备:** 如果手机开机运行了较长时间，可能会导致刷入失败。尝试重启手机后立即再次刷入。
    *   **更换脚本:** 如果一个版本的模块加载失败，可以尝试同一内核版本下的其他分支 (例如，如果 `GT/5.10/.../load.sh` 失败，可尝试 `RT/5.10/.../load.sh` 或 `rt proc/5.10/.../load.sh`)，但务必确保内核版本号匹配。
    *   **重启再试:** 大部分失败提示都建议重启手机后再尝试。

如果多次尝试后依然失败，建议停止操作，并在相关技术社区或群组中寻求帮助，同时提供详细的设备信息、内核版本以及尝试过的操作和错误日志。

## 重要提示与警告 (Important Notes and Warnings)

*   **ROOT权限是前提:** 再次强调，所有操作都需要您的设备已经获取了 ROOT 权限。没有 ROOT 权限，`insmod` 命令将无法执行，模块无法加载。
*   **兼容性风险:** 内核模块与特定的内核版本和配置紧密相关。刷入不兼容的模块极有可能导致系统不稳定、应用程序崩溃、设备无限重启 (bootloop) 或其他未知问题。请务必仔细核对内核版本。
*   **谨慎操作，后果自负:** 您需要对自己的操作负责。如果您不完全理解这些操作的含义和潜在风险，请不要轻易尝试。建议在操作前备份重要数据。
*   **遵循原始说明:** 项目中包含的 `说明(第一次刷驱动必看...).txt` 文件提到，如果执行sh后不是直接结束进程，而是有英文提示，则为报错。如果直接结束进程则为成功。这是判断成功与否的一个基本标准。
*   **`王者荣耀.sh` 脚本:** 根目录下的 `王者荣耀.sh` 文件表明这些模块可能被用于优化该游戏。具体功能和使用方法需要用户自行查看该脚本内容。使用此类针对特定应用的脚本时，也请注意其来源和潜在影响。
*   **进程隐藏功能:** 名称中带有 `(新方案隐藏进程)` 的模块具备隐藏进程的能力。请合规使用此功能，不要用于非法用途。
*   **官方社群:** 原始说明文件提到 "请发到官方群里" 以报告错误。如果存在这样的社群，加入并寻求帮助可能是解决问题的好方法。
*   **非官方修改:** 这些模块属于对系统底层的修改，可能不被设备制造商或游戏运营商所支持。使用这些模块可能带来的任何后果（例如账户问题）需用户自行承担。

**安全操作建议:**
1.  在进行任何修改前，尽可能备份您的设备数据。
2.  从小处着手，例如先尝试一个您认为最匹配且风险较低的模块。
3.  如果遇到问题，立即停止进一步操作，并尝试通过重启恢复。如果无法恢复，您可能需要通过 Recovery 模式刷入官方固件或寻求专业帮助。
